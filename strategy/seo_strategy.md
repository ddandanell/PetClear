## 1. Executive Summary & Strategic Positioning

### Core Challenge
New domain, zero authority, entering a competitive market with established players (ISS Relocations, Pet Express UAE, Pawpaths, Paws & Planes). Direct competition for head terms like "pet relocation Dubai" is not viable for 6–12 months.

### Strategic Approach: The Topical Authority Ladder

| Phase | Timeline | Focus | Target Keywords |
|-------|----------|-------|-----------------|
| **Foundation** | Months 1–3 | Long-tail informational, route-specific, FAQ | Low competition, high intent |
| **Authority** | Months 4–6 | Pillar content expansion, breed guides, local SEO | Medium competition |
| **Expansion** | Months 7–12 | Head terms, programmatic SEO, Arabic content | High competition |

### Key Differentiators to Signal in SEO
1. **Radical pricing transparency** — competitors hide costs; we publish breakdowns
2. **Pet-specialist expertise** — not a general mover with a pet add-on
3. **Digital-first experience** — WhatsApp, instant quotes, tracking
4. **Emotional authenticity** — founder story, real pet photos, reunion moments
5. **Regulatory depth** — MOCCAE, IATA, breed-specific accuracy

### E-E-A-T Priorities for a New Domain
- **Experience:** Founder relocated with pet to Dubai (first-hand narrative)
- **Expertise:** IPATA/IATA credentials, veterinary partnerships, MOCCAE licensing
- **Authoritativeness:** Guest posts on expat forums, citations in relocation guides
- **Trustworthiness:** Transparent pricing, visible reviews, clear policies, HTTPS, privacy policy

## 3. Schema Markup Strategy

### Implementation Philosophy
All schema markup must be delivered as **JSON-LD** in the `<head>` of each page. Validate all implementations using Google's [Rich Results Test](https://search.google.com/test/rich-results) before publishing. For a new domain, schema markup is a **critical differentiator** — most competitors have minimal or no structured data.

## 5. Technical SEO Foundation

### 5.1 URL Structure Rules

| Rule | Example | Rationale |
|------|---------|-----------|
| Lowercase only | `/guides/uae-pet-import-requirements/` | Consistency, no duplicate content |
| Hyphens for spaces | `/pet-relocation-cost-dubai/` | Google treats hyphens as word separators |
| No trailing slash inconsistency | All URLs use trailing slash OR none | Pick one, redirect the other |
| No dates in URLs | `/guides/cost-guide/` NOT `/guides/2026/06/cost-guide/` | Evergreen content, no annual 301s needed |
| No unnecessary parameters | `/services/` NOT `/services/?ref=seo` | Clean crawl paths |
| Keyword in URL | `/routes/uk-to-dubai/` | Modest ranking signal, user clarity |
| Maximum 3 folder levels | `/routes/uk-to-dubai/` = 2 levels | Crawl efficiency |
| No underscores | `/uk-to-dubai/` NOT `/uk_to_dubai/` | Underscores are not word separators |

**V1 URL Patterns:**
```
/
/services/
/guides/[topic-slug]/
/routes/[origin]-to-[destination]/
/about/
```

**Future URL Patterns:**
```
/blog/[year]/[month]/[post-slug]/
/breeds/[breed-name]/
/routes/[origin]-to-[destination]/
/local/[dubai-neighborhood]/
/ar/... (Arabic mirror)
```

### 5.2 Canonicalization Strategy

| Scenario | Canonical URL | Implementation |
|----------|-------------|----------------|
| Homepage | `https://www.dubaipetrelo.com/` | Self-canonical + redirect `/index.html` |
| HTTP -> HTTPS | `https://...` | 301 redirect all HTTP |
| www vs. non-www | `https://www.dubaipetrelo.com/...` | 301 redirect non-www to www (or vice versa) |
| Trailing slash | Consistent choice | 301 redirect `/page` -> `/page/` (or reverse) |
| URL parameters | Strip params | `?utm_source=...` -> canonical without params |
| Arabic version | `https://www.dubaipetrelo.com/ar/...` | `hreflang` + self-canonical |
| Print-friendly | Original page | Print CSS, no separate URL |

### 5.5 Core Web Vitals Targets

| Metric | Target | Good | Needs Improvement | Poor |
|--------|--------|------|-------------------|------|

**Implementation Tactics:**
- **CDN:** Use Cloudflare or AWS CloudFront (server in UAE or nearest edge)
- **Images:** WebP format, lazy loading, responsive `srcset`, max 200KB per hero image
- **Fonts:** `font-display: swap`, preload critical fonts, subset for Latin + Arabic
- **JavaScript:** Defer non-critical JS, inline critical CSS, code-split by route
- **Server:** Enable Brotli compression, HTTP/2, cache-control headers
- **Third-party:** Load Google Analytics, chat widgets, review widgets asynchronously

| **Dimensions** | Hero: 1920x800 (max), Content: 1200x800, Thumbnails: 400x300 |
| **File Size** | Hero <= 200KB, Content <= 100KB, Thumbnails <= 50KB |

| **Structured Data** | `ImageObject` schema with width, height, caption |
